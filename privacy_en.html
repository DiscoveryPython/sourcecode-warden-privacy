Source Code Warden Privacy Policy

Last updated: April 2026

1. Information Collection

Source Code Warden does not collect, store, or transmit personal information to the developer.

2. Local File and Folder Access

Source Code Warden may ask you to select local files, project folders, and backup target folders so that you can create safe, traceable backups on your device. Any files or folders you choose are used only for local backup operations and are not uploaded to the developer.

3. Backup Processing

Source Code Warden is designed to process your selected source files and folders locally on your Mac. Backup queue data, target folder assignments, naming schemes, generated backup names, progress information, and activity logs are handled on-device.

If the app creates temporary working data during backup processing, that data is used only to complete the requested backup operation on your device.

4. Network Access

Source Code Warden does not require developer-operated servers for its core backup features. Backup creation, folder assignment, naming, and activity logging are performed locally on your Mac.

If any future feature involves opening a user-provided online resource or external link, your device would connect directly to the relevant third-party service, not through the developer's server.

Third-party services may receive technical information such as your IP address as part of standard network communication. Their handling of information is governed by their own privacy policies.

5. Third-Party SDKs

Source Code Warden does not include third-party analytics or advertising SDKs.

6. Data Security

Your source files, project folders, backup files, target folder paths, and activity logs remain on your device unless you choose to save, move, copy, or share them yourself. Source Code Warden does not upload your source code or backup data to the developer.
