Welcome to Source Code Warden! If you encounter any issues while selecting source files, assigning backup folders, choosing a naming scheme, or creating backups, please check the FAQ below first.

Frequently Asked Questions (FAQ)

Q: What can Source Code Warden do?

A: Source Code Warden helps you select source files or project folders, assign backup target folders, choose a backup naming scheme, and create safe, traceable backup copies.

Q: Why can’t the app access my files or backup folder?

A: Please make sure you granted file or folder access permissions, and try selecting the source files, project folder, or target folder again if needed.

Q: Why does a Normal Backup fail when a backup already exists?

A: Normal Backup uses a strict daily name. If a backup with the same generated name already exists, choose a different target folder, remove the existing backup, or use Debug Version Backup to create the next available version suffix.

Q: What is Debug Version Backup?

A: Debug Version Backup scans the target folder and automatically creates the next available suffix, such as A, B, C, or AA, so existing backup files are not overwritten.

Q: How can I report a bug or request a feature?

A: Contact us by email or submit an issue via GitHub.
